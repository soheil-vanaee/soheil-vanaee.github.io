case study — real estate platform

Estate Nove — Real Estate Platform

A fully responsive real-estate marketplace — home, property search & filtering, property details, a mortgage calculator, and full authentication flows — designed end-to-end in Figma and built with React for a consistent, fully working experience, not just a mockup.

role

UI/UX design + React front-end development

timeline

Personal project — 2026

tools

Figma, React, React Router, Vite

tags

case study, real estate, react development

overview

The problem

Most real-estate sites are either overloaded with filters or too generic to feel trustworthy. The goal was a browsing experience that stays clean and fast while still giving buyers, renters, and sellers a clear, short path from "just looking" to "talking to an agent" — with every step of that path actually working, not just designed.

This is a self-directed case study: there was no client or user research pool behind it. I set the brief myself, based on patterns I noticed while benchmarking real-estate platforms, and used it as an end-to-end exercise in owning both the design and the build.

process

UX approach

I designed the full site as one connected system in Figma — spacing, typography, color, and component patterns stayed consistent from the landing page through search, property details, and checkout-adjacent flows like contacting an agent or applying for financing.

  • Benchmarked common real-estate platform patterns: listing cards, faceted filters, property detail layouts
  • Designed a token-based visual system (navy + bronze palette, type scale, spacing) reused across every screen
  • Wireframed and then designed high-fidelity screens for home, listings, property details, mortgage calculator, and auth
  • Applied progressive disclosure to filters ("+ more option") to keep the default search bar uncluttered
  • Designed inline validation and empty states for every form, so the interface never fails silently
  • Built the entire system in React with real routing, so every tab, card, and footer link leads to a real page
user flow

From landing page to agent contact

The flow takes a visitor from the homepage to browsing and filtering listings, into a property's full details, and out to either a mortgage estimate or a direct conversation with an agent — with sign-up available but never required to get there.

outcome

Result & learnings

The result is a fully working, responsive React application: 13 real routes (not static mockups), including listings filtered by type, a live mortgage calculator, and validated auth forms. Owning both design and development end-to-end forced tighter decisions — every visual choice had to survive being actually built, and every component had to earn its place in the design system rather than being a one-off screen.

The biggest lesson was around scope: designing and building solo means the design system is the plan — investing early in tokens and reusable components (search filter, property card, listing page shell) made adding new pages like Home Loans or Sell dramatically faster later on.
