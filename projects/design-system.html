UX approach

I treated motion as a first‑class part of the system, not a final polish step, defining it alongside each component's states.

  • Audited existing screens and consolidated repeated patterns into components
  • Built variables and Auto Layout structures for consistent, scalable components
  • Defined component variants and interaction states (default, hover, active, disabled)
  • Designed and documented motion using Figma Motion for key interactions
  • Piloted the system on a real feature before rolling it out team‑wide
